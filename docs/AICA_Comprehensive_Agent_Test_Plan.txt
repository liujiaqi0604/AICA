# AICA Agent 综合功能测试计划

## 测试说明

本测试计划专为验证 AICA Agent 的综合能力而设计。测试场景全部为**综合类测试**，模拟真实开发场景，需调用多个工具协同完成任务。

### 测试目标
- 验证 Agent 的多步骤任务规划与执行能力
- 验证工具选择的准确性与参数传递的正确性
- 验证复杂场景下的上下文保持与状态管理
- 覆盖全部 13 个 AICA 工具

### 测试方法
1. **我制定测试方案**（当前文档）
2. **你使用 AICA 执行测试** 并在每个测试用例后记录实际结果
3. **我验证测试结果的正确性** 并输出分析报告

### 评分标准
| 评级 | 说明 |
|------|------|
| **通过** | 工具链选择正确、参数准确、任务完成、无幻觉 |
| **部分通过** | 主要目标达成，但有冗余调用、参数小错误或需人工干预 |
| **失败** | 工具选择错误、关键参数错误、任务未完成、产生幻觉 |

## 发现的问题与修复建议

### 问题1: 强制完成机制过早触发

**问题描述**:
- AgentExecutor.cs 第121-126行的强制完成逻辑在第6轮迭代时就触发
- 条件：`_taskState.Iteration >= 6` 导致复杂任务无法完成
- 场景一需要至少8-10轮才能完成所有子任务

**影响范围**: 所有需要多步骤探索的任务

**建议修复方案**:
```csharp
// 修改 AgentExecutor.cs 第121-126行
bool forceCompletion = _taskState.HasEverUsedTools &&
    (_taskState.TotalToolCallCount >= 12 ||        // 从8改为12
     _taskState.Iteration >= 10 ||                 // 从6改为10
     tokenUsageRatio > 0.70 ||                     // 从0.60改为0.70
     _taskState.Iteration >= _maxIterations - 2);
```

**理由**:
- 综合测试场景通常需要8-12轮迭代
- 过早强制完成会导致任务未完成就被截断
- 应该优先依赖 `attempt_completion` 工具的主动调用

---

### 问题2: 强制完成时移除工具定义导致无法调用 attempt_completion

**问题描述**:
- AgentExecutor.cs 第184行：`var effectiveTools = forceCompletion ? null : toolDefinitions;`
- 强制完成时传递 `null` 工具列表，导致LLM无法调用 `attempt_completion`
- 这违背了 `attempt_completion` 工具的设计初衷（MUST USE）

**影响范围**: 所有被强制完成的任务

**建议修复方案**:
```csharp
// 修改 AgentExecutor.cs 第184行
// 强制完成时仍保留 attempt_completion 工具
var effectiveTools = forceCompletion
    ? toolDefinitions.Where(t => t.Name == "attempt_completion").ToList()
    : toolDefinitions;
```

**理由**:
- 即使强制完成，也应该让Agent通过 `attempt_completion` 正常结束
- 这样可以保证UI渲染的一致性（始终有completion卡片）
- 符合工具设计的语义（CRITICAL - MUST USE）

---

### 问题3: UI渲染异常 - 工具调用框消失

**问题描述**:
- 流式输出时：蓝色工具调用缩放框正常显示
- 最终渲染时：工具调用框消失，只剩下文本
- 可能原因：ChatToolWindowControl.xaml.cs 的消息持久化逻辑有问题

**需要检查的代码位置**:
1. ChatToolWindowControl.xaml.cs 第700-703行：最终内容的构建逻辑
2. 第337-343行和411-417行：CompletionResult的渲染逻辑
3. 消息持久化时是否正确保存了 ToolLogsHtml

**建议排查步骤**:
1. 检查 `finalToolLogs` 是否正确传递到消息对象
2. 检查强制完成时 `hasToolCalls` 标志是否正确
3. 验证 `RenderFinalAssistantMessage` 方法的调用时机

---

### 问题4: 输出格式不清晰

**问题描述**:
- 期望格式：工具调用框（可折叠） + 流式文本 + completion卡片
- 实际格式：混乱，缺少清晰的分隔和结构

**可能原因**:
- 强制完成时的 `[SUMMARIZE]` 系统消息导致LLM输出格式混乱
- UI层没有正确处理强制完成的特殊情况

**建议优化**:

2. UI层对强制完成的消息做特殊标记和渲染
