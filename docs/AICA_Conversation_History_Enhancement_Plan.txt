一、概述

================================================================================
二、需求详情
================================================================================

需求 1: 会话列表删除按钮右对齐
----------------------------------------
【优先级】: 高
【类型】: UI 优化

【当前问题】:
- 会话列表中每个会话项的删除按钮（🗑）位置不统一
- 删除按钮紧跟在会话标题后面，视觉效果不佳

【期望效果】:
- 所有删除按钮在会话列表右侧垂直对齐
- 无论会话标题长短，删除按钮始终在最右侧
- 提升 UI 的整洁度和专业感

【实现方案】:
1. 修改 ChatToolWindowControl.xaml 中的 ListBox.ItemTemplate
2. 调整 Grid 布局，确保删除按钮列宽度为 Auto，且 HorizontalAlignment="Right"
3. 可能需要调整会话标题列的宽度为 "*"（占据剩余空间）

【涉及文件】:
- src\AICA.VSIX\ToolWindows\ChatToolWindowControl.xaml


需求 2: 自动切换项目会话列表
----------------------------------------
【优先级】: 高
【类型】: 功能增强

【当前问题】:
- 用户打开新项目/解决方案时，会话历史列表不会自动更新
- 需要手动关闭并重新打开侧边栏才能看到新项目的会话

【期望效果】:
- 当用户在 Visual Studio 中打开新的项目或解决方案时
- AICA 插件自动检测到项目切换
- 自动刷新会话历史列表，显示新项目的会话记录
- 如果侧边栏已打开，列表自动更新；如果未打开，下次打开时显示正确的列表

【实现方案】:
1. 监听 Visual Studio 的解决方案/项目打开事件
   - 使用 DTE2.Events.SolutionEvents.Opened
   - 使用 DTE2.Events.SolutionEvents.AfterClosing
2. 在事件处理器中：
   - 保存当前会话（如果有）
   - 清空当前会话 ID
   - 如果侧边栏已打开，调用 LoadConversationListAsync() 刷新列表
3. 在 ChatToolWindowControl 构造函数或 Loaded 事件中注册这些事件
4. 在 Unloaded 事件中取消注册，避免内存泄漏

【涉及文件】:
- src\AICA.VSIX\ToolWindows\ChatToolWindowControl.xaml.cs

【技术细节】:
- 需要添加 _solutionEvents 字段存储事件引用
- 事件处理需要在 UI 线程上执行（使用 ThreadHelper.JoinableTaskFactory.SwitchToMainThreadAsync）


需求 3: 支持 "/allresume" 显示所有会话
----------------------------------------
【优先级】: 中
【类型】: 功能增强

【当前问题】:
- 会话历史列表只显示当前项目的会话
- 用户无法查看其他项目的历史会话
- 缺少跨项目查看会话的能力

【期望效果】:
- 用户在搜索框中输入 "/allresume" 时
- 会话列表显示 .AICA 目录下所有项目的所有会话记录
- 每个会话项显示所属项目名称（便于区分）
- 输入其他内容时恢复正常的项目过滤和搜索功能

【实现方案】:
1. 修改 SearchTextBox_TextChanged 事件处理器
2. 检测输入是否为 "/allresume"（不区分大小写）
3. 如果是 "/allresume"：
   - 调用 _conversationStorage.ListAllConversationsAsync() 获取所有会话
   - 在 ConversationViewModel 中添加 ProjectName 字段
   - 修改 XAML 模板，在会话标题下方显示项目名称（小字、灰色）
4. 如果不是 "/allresume"：
   - 恢复原有逻辑（项目过滤 + 标题搜索）
5. 需要在 ConversationStorage 中添加 ListAllConversationsAsync() 方法

【涉及文件】:
- src\AICA.VSIX\ToolWindows\ChatToolWindowControl.xaml.cs
- src\AICA.VSIX\ToolWindows\ChatToolWindowControl.xaml
- src\AICA.Core\Storage\ConversationStorage.cs

【UI 变化】:
会话项显示格式（/allresume 模式）：
┌─────────────────────────────────┐
│ 会话标题                    🗑  │
│ 项目名称 | 5分钟前              │
└─────────────────────────────────┘

会话项显示格式（正常模式）：
┌─────────────────────────────────┐
│ 会话标题                    🗑  │
│ 5分钟前                         │
└─────────────────────────────────┘

================================================================================
五、技术风险与注意事项
================================================================================

1. 需求 2 的事件监听可能导致内存泄漏
   - 必须在 Unloaded 时正确取消注册事件
   - 使用弱引用或确保事件处理器生命周期正确

2. 需求 2 的线程安全问题
   - DTE 事件可能在后台线程触发
   - 必须使用 ThreadHelper 切换到 UI 线程

3. 需求 3 的性能问题
   - 如果会话数量很多，加载所有会话可能较慢
   - 考虑添加分页或限制显示数量（如最近 100 个）

4. 需求 3 的 UI 适配
   - 需要确保项目名称显示不会导致会话项过高
   - 考虑使用 Tooltip 显示完整项目路径
