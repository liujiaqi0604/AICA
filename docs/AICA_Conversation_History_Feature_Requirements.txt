AICA 会话历史管理功能需求文档
================================================================================

文档版本: v1.0
创建日期: 2026-03-10
项目名称: AICA (AI Coding Assistant)
需求类型: 新功能开发

================================================================================
一、需求概述
================================================================================

为 AICA 添加完整的会话历史管理功能，使用户能够保存、浏览、恢复和管理与 AI 的对话记录。
该功能需要支持项目级别的会话隔离，确保不同项目的会话记录互不干扰。

================================================================================
二、核心功能需求
================================================================================

2.1 会话持久化
--------------
【需求描述】
- AICA 需要自动保存用户与 AI 的完整对话历史
- 会话数据包括：用户消息、AI 回复、工具调用记录、时间戳等
- 支持会话的增量保存（每次对话后自动保存）

【技术要求】
- 存储格式：JSON
- 存储位置：C:\Users\{用户名}\.AICA\conversations\
- 文件命名：{conversationId}.json
- 编码格式：UTF-8

【数据结构】
{
  "id": "会话唯一标识符",
  "title": "会话标题（取自第一条用户消息）",
  "projectPath": "项目根目录路径（用于项目隔离）",
  "projectName": "项目名称",
  "solutionPath": "解决方案文件路径",
  "workingDirectory": "工作目录",
  "createdAt": "创建时间（ISO 8601）",
  "updatedAt": "最后更新时间（ISO 8601）",
  "messages": [
    {
      "role": "user|assistant",
      "content": "消息内容",
      "toolLogsHtml": "工具调用日志HTML（仅assistant）",
      "completionData": "完成数据（仅attempt_completion）",
      "timestamp": "消息时间戳"
    }
  ]
}

2.2 会话恢复
--------------
【需求描述】
- 用户可以从历史记录中选择任意会话并恢复
- 恢复后，聊天界面显示完整的历史对话
- 用户可以在恢复的会话基础上继续对话
- AI 能够访问完整的上下文历史

【功能要点】
- 恢复会话时，清空当前会话
- 加载历史消息到 UI 界面
- 加载历史消息到 LLM 上下文（_llmHistory）
- 保持工具调用日志的格式和样式
- 恢复后的新消息继续追加到同一会话文件

2.4 项目级别会话隔离
------------------
【需求描述】
- 会话记录与 Visual Studio 打开的项目/解决方案关联
- 切换项目时，会话列表只显示当前项目的历史记录
- 不同项目的会话互不干扰

【实现方案】
1. 项目识别：
   - 优先使用解决方案文件路径（.sln）作为项目标识
   - 如果没有解决方案，使用工作目录路径
   - 标准化路径（统一大小写、斜杠方向）

2. 会话过滤：
   - 加载会话列表时，只显示 projectPath 匹配当前项目的会话
   - 如果用户未打开任何项目，显示所有无项目关联的会话

3. 项目信息存储：
   - projectPath: 项目根目录绝对路径
   - projectName: 从解决方案文件或目录名提取
   - solutionPath: 解决方案文件完整路径（如果存在）

【边界情况处理】
- 项目路径变更：如果项目移动到新位置，会话将无法自动关联
  → 解决方案：提供"会话迁移"功能（可选，后期优化）
- 多解决方案项目：如果一个目录下有多个 .sln 文件
  → 使用当前打开的解决方案路径作为标识

2.5 自动保存机制
--------------
【需求描述】
会话数据需要在以下时机自动保存，无需用户手动操作。

【保存时机】
1. 每次对话完成后（AI 回复结束）
2. 用户切换到其他会话时
3. AICA 窗口关闭时
4. Visual Studio 退出时
5. 用户点击"新建会话"时

【保存策略】
- 增量保存：只保存变更的会话
- 异步保存：不阻塞 UI 线程
- 错误处理：保存失败时记录日志，不影响用户操作
- 自动清理：保留最近 100 个会话，超出部分自动删除旧会话

【性能优化】
- 使用防抖机制：连续保存请求合并为一次
- 延迟保存：对话结束后延迟 500ms 保存（避免频繁 IO）

2.6 会话删除功能
--------------
【需求描述】
用户可以删除不需要的会话记录。

【功能要点】
1. 删除入口：会话列表中每个会话项右侧的 [🗑] 按钮
2. 确认机制：点击删除按钮后弹出确认对话框
3. 删除操作：
   - 从磁盘删除对应的 JSON 文件
   - 从会话列表中移除该项
   - 如果删除的是当前激活会话，自动创建新会话
4. 批量删除（可选）：支持选中多个会话批量删除

【安全措施】
- 删除操作不可撤销，必须二次确认
- 删除失败时显示错误提示
- 当前正在使用的会话删除时，提示"是否确定删除当前会话？"

三、非功能性需求
================================================================================

3.1 性能要求
--------------
- 会话列表加载时间：< 200ms（100个会话）
- 会话恢复时间：< 500ms（包含 UI 渲染）
- 会话保存时间：< 100ms（异步，不阻塞 UI）
- 侧边栏展开/收起动画：300ms

3.2 存储要求
--------------
- 单个会话文件大小：通常 < 1MB
- 最大会话保留数量：100 个（可配置）
- 存储目录：C:\Users\{用户名}\.AICA\conversations\
- 自动创建目录：如果目录不存在，自动创建

3.3 兼容性要求
--------------
- Visual Studio 版本：2022 及以上
- Windows 版本：Windows 10/11
- .NET Framework：4.8
- 文件系统：NTFS

3.4 安全性要求
--------------
- 会话文件权限：仅当前用户可读写
- 敏感信息处理：不存储 API Key 等敏感配置
- 路径安全：防止路径遍历攻击

3.5 可维护性要求
--------------
- 代码模块化：会话管理功能独立模块
- 日志记录：关键操作记录日志（保存、加载、删除）
- 错误处理：所有异常捕获并记录，不崩溃
- 单元测试：核心功能覆盖率 > 80%

```csharp
private string GetCurrentProjectPath()
{
    ThreadHelper.ThrowIfNotOnUIThread();

    // 优先使用解决方案路径
    if (_dte?.Solution?.FullName != null && !string.IsNullOrEmpty(_dte.Solution.FullName))
    {
        return Path.GetDirectoryName(_dte.Solution.FullName);
    }

    // 回退到工作目录
    return _agentContext?.WorkingDirectory;
}

private string GetCurrentSolutionPath()
{
    ThreadHelper.ThrowIfNotOnUIThread();
    return _dte?.Solution?.FullName;
}

private string GetCurrentProjectName()
{
    ThreadHelper.ThrowIfNotOnUIThread();

    if (_dte?.Solution?.FullName != null)
    {
        return Path.GetFileNameWithoutExtension(_dte.Solution.FullName);
    }

    if (_agentContext?.WorkingDirectory != null)
    {
        return Path.GetFileName(_agentContext.WorkingDirectory);
    }

    return "未命名项目";
}
```

7.2 会话过滤
--------------
```csharp
public async Task<List<ConversationSummary>> ListConversationsForProjectAsync(
    string projectPath,
    int limit = 50)
{
    var allConversations = await ListConversationsAsync(limit * 2);

    if (string.IsNullOrEmpty(projectPath))
    {
        // 无项目时，返回所有无项目关联的会话
        return allConversations
            .Where(c => string.IsNullOrEmpty(c.WorkingDirectory))
            .Take(limit)
            .ToList();
    }

    // 标准化路径（统一大小写和斜杠）
    var normalizedPath = NormalizePath(projectPath);

    return allConversations
        .Where(c => !string.IsNullOrEmpty(c.WorkingDirectory) &&
                    NormalizePath(c.WorkingDirectory) == normalizedPath)
        .Take(limit)
        .ToList();
}

private string NormalizePath(string path)
{
    if (string.IsNullOrEmpty(path)) return string.Empty;
    return Path.GetFullPath(path).ToLowerInvariant().Replace('/', '\\');
}
```

7.3 会话恢复
--------------
```csharp
private async Task LoadConversationAsync(string conversationId)
{
    var record = await _conversationStorage.LoadConversationAsync(conversationId);
    if (record == null)
    {
        ShowError("无法加载会话");
        return;
    }

    // 清空当前会话
    _conversation.Clear();
    _llmHistory.Clear();

    // 恢复消息
    foreach (var msg in record.Messages)
    {
        _conversation.Add(new ConversationMessage
        {
            Role = msg.Role,
            Content = msg.Content,
            ToolLogsHtml = msg.ToolLogsHtml,
            CompletionData = msg.CompletionData
        });

        // 恢复到 LLM 历史（用于上下文）
        if (msg.Role == "user")
        {
            _llmHistory.Add(ChatMessage.User(msg.Content));
        }
        else if (msg.Role == "assistant")
        {
            _llmHistory.Add(ChatMessage.Assistant(msg.Content));
        }
    }

    // 设置当前会话 ID
    _currentConversationId = conversationId;

    // 重新渲染界面
    RenderConversation();
}
```

7.4 自动保存增强
--------------
```csharp
// 在 ChatToolWindowControl 构造函数中
public ChatToolWindowControl()
{
    InitializeComponent();

    // 监听窗口卸载事件
    this.Unloaded += OnWindowUnloaded;
}

private async void OnWindowUnloaded(object sender, RoutedEventArgs e)
{
    await SaveConversationAsync();
}

// 在 ChatToolWindow 中监听 VS 退出
protected override void Initialize()
{
    base.Initialize();

    ThreadHelper.ThrowIfNotOnUIThread();
    var dte = (DTE2)GetService(typeof(DTE));
    if (dte != null)
    {
        dte.Events.DTEEvents.OnBeginShutdown += OnVSShutdown;
    }
}

private void OnVSShutdown()
{
    ThreadHelper.JoinableTaskFactory.Run(async () =>
    {
        var control = Content as ChatToolWindowControl;
        await control?.SaveConversationAsync();
    });
}
```

九、风险与挑战
================================================================================

9.1 技术风险
--------------
风险：WPF 侧边栏动画可能影响性能
缓解：使用硬件加速，优化动画实现

风险：大量会话导致列表加载缓慢
缓解：实现虚拟化列表，分页加载

风险：项目路径变更导致会话丢失
缓解：后期提供会话迁移功能

9.2 用户体验风险
--------------
风险：用户可能不理解项目隔离机制
缓解：在 UI 中明确提示当前项目名称

风险：误删重要会话
缓解：二次确认，考虑添加回收站功能
